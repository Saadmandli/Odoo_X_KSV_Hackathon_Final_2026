## Deployment

**Split across three free tiers, no credit card:**

| Piece | Host | Why |
|---|---|---|
| Postgres | **Neon** | Free forever. Render's free Postgres **expires after 30 days** |
| API | **Render** (web service, free) | Long-lived Node process — no serverless cold-start-per-request, and Socket.IO stays possible later |
| Frontend | **Vercel** (static build) | Free, instant, never sleeps |

### Steps

1. **Neon** — create project, copy the **pooled** connection string (the host
   containing `-pooler`). Prisma opens many connections; the direct string
   exhausts them.
2. **Seed production from your laptop.** Render's free tier has no shell, so:
   ```bash
   cd server
   CARPOOL_DATABASE_URL="<neon-pooled-url>" npm run db:seed
   ```
3. **Render** — New > Blueprint, point at this repo. `render.yaml` provisions
   the service. Set in the dashboard:
   - `CARPOOL_DATABASE_URL` = Neon pooled string
   - `CORS_ORIGINS` = the Vercel URL (once you have it)
4. **Vercel** — import repo, root directory `client`. Set `VITE_API_URL` to the
   Render URL **before the first build** — `VITE_` vars are inlined at build
   time, so a bundle built without it silently calls `localhost`.
5. **Keep the API awake.** Free Render services sleep after 15 min idle and cold
   start takes ~50s. Point UptimeRobot or cron-job.org at
   `https://<api>.onrender.com/api/health` every 10 minutes. 744 hrs/month sits
   under the 750-hour free cap — but only for **one** service, which is why the
   frontend is on Vercel and not a second Render service.

### Known deploy tripwires

- `prisma migrate deploy` runs in Render's build (in `render.yaml`) — verified
  against an empty database, creates all 12 tables.
- `JWT_SECRET` is generated by Render. Rotating it logs everyone out; don't
  redeploy with a new one mid-demo.
- CORS must list the exact Vercel origin, no trailing slash.

## Demo script (rehearse this)

Two browsers side by side: Prayag (driver) and Saad (passenger).

1. Saad: Find a ride, Bopal to GIFT City, **route confirmation shows real distance**
2. Saad: Book a seat on Prayag's ride
3. Prayag: notification appears; My Trips shows the passenger
4. Prayag: **Start trip** — marker moves on the live map, ETA counts down
5. Saad: watches the same trip move in real time
6. Saad: shares the live link; open it in a third tab with no login
7. Prayag: Complete trip
8. Saad: pay by wallet, balance drops and Prayag's rises
9. Both: trip lands in Ride History
10. Prayag: **Reports** — fuel consumed, cost per km, efficiency trend
11. Shrey (admin): dashboard, revoke an employee's access, show them locked out

Steps 4 to 6 are the money shot. Rehearse them until they're boring.
